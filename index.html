RNN

Recurrent Neural Network

  1. 普通的NN 一个输入对应一个输出
  2. 一对多 ： 如image - > sequcence of words(图片对应一句话描述)
  3. 多对一 : 如视频识别标签
  4. 多对多 ：视频描述

RNN的内部状态(隐藏状态) 在每一次输入都进行更新

h_t(new_state) = fW(h_t-1, x_t)
y_t = fW_hy(h_t) fW_hy-> 将hidden state转化为输出的output

更新的fW 和 转化的fW_hy 都用一致的函数

Vanilla（普通）RNN

用tanh的好处是可以吧ht的值压在(-1,1)之间

RNN 的 计算图(computational graph)

每一步的Loss 相加得到总Loss，对每个时间步Loss都能得到对W的梯度

对于多对一，可能只对最后的hidden state相关，所以只用考虑最后的Loss

$$\delta_t = \frac{\partial L_t}{\partial s_t} + \frac{\partial s_{t+1}}{\partial s_t} \delta_{t+1}$$

用这条公式可以计算所有的L，其中$\delta_t$是总L 对 s_t的导数

问题:当时间序列长的时候，每一个时间步的反向传播都异常艰难

loca
解决:随时间反向传播

（多输出）我们分割时间窗，然后只看这个时间窗里面的Loss，之前的只看成为h_0

（单输出） 前向传播和原来一样，反向传播从后往前分为不同的时间窗，每个时间窗给前面的只有Loss对x1和对前一个隐藏状态的导数

如果每层W的不一样 -> 失去递推性，相当于一个普通的NN了

语言RNN模型: 将前一个状态softmax后的输出，作为第二个的输入，同时hidden layers也在由于输入在改变，同时前一个状态的输出是根据softmax输出的概率采样得到

缺点：

  1. recurrent 反向传播（每一步都需要前面的hidden states）很慢
  2. 失去前面的信息

RNN在CV的应用：

  1. 图片说明文字 h_0 为 CNN后的图片，序列输入为前一步预测的词元
  1. Visual Question Answering

LSTM

RNN梯度计算 ->

十分容易出现梯度消失，所以我们用LSTM

LSTM : (i, f, o, g) (sigmoid, sigmoid, sigmoid ,tanh) W(ht_1, x)

每个门都有自己独立的一套权重矩阵 $W$ 和偏置 $b$。它们的计算公式高度一致，只有激活函数和物理意义不同。

i,f,o由于是sigmoid，输出在(0, 1 )之间

$$C_t = f_t \odot C_{t-1} + i_t \odot \tilde{C}_t$$

f 控制之前的状态有多少要被记住， i控制新状态多少要被输入

$$h_t = o_t \odot \tanh(C_t)$$

o控制输出的状态和原来状态的关系

由于C_t 到 C_t-1 之间不经过tanh函数，所以我们不用担心梯度消失

和ResNet的思想很相似,跳过一些层

Modern RNNs(MAMBA)

  1. 上下文没有限制 (相比Transformer的优势)
  2. 前向传播的计算增长是线性的， 不需要遍历整个序列

测试用例的答案是一个 32-位 整数。

请注意，一个只包含一个元素的数组的乘积是这个元素的值。

示例 1:

输出: 6

示例 2:

输入: nums = [-2,0,-1]
输出: 0
解释: 结果不能为 2, 因为 [-2,-1] 不是子数组。

以0为间隔，左右双指针找到最近的负数

    整数数组 nums 按升序排列，数组中的值 互不相同 。

在传递给函数之前，nums 在预先未知的某个下标 k（0 <= k < nums.length）上进行了 向左旋转，使数组变为 [nums[k], nums[k+1], …, nums[n-1], nums[0], nums[1], …, nums[k-1]]（下标 从 0 开始 计数）。例如， [0,1,2,4,5,6,7] 下标 3 上向左旋转后可能变为 [4,5,6,7,0,1,2] 。

给你 旋转后 的数组 nums 和一个整数 target ，如果 nums 中存在这个目标值 target ，则返回它的下标，否则返回 -1 。

你必须设计一个时间复杂度为 O(log n) 的算法解决此问题

class Solution:
    def search(self, nums: List[int], target: int) -> int:
        # 两个递增 且一个递增比第二个小
        k = nums[0]
        N = len(nums)
        left, right = 0, N - 1
        while left <= right:
            mid = (left + right) // 2
            if nums[mid] == target:
                return mid
            if nums[left] <= nums[right]:
                if nums[mid] >= target:
                    right = mid - 1
                else:
                    left = mid + 1
            else:
                if (target >= k) == (nums[mid] >= k):
                    if nums[mid] <= target:
                        left = mid + 1
                    else:
                        right = mid - 1
                else:
                    if nums[mid] >= target:
                        left = mid + 1
                    else:
                        right = mid - 1
        return -1

Lecture 6: CNN Architectures

CNN Architectures

Normalization Layers

根据学习到参数的均值和标准差来缩放和平移我们的input data(输入数据)

问题：怎么计算均值和标准差

x : N * D

从D方向上：计算μ，σ ： N * 1 (per batch)

scale因子 : 从N方向计算 ： 1 * D

不同的Norm形式

Batch Normalization (BatchNorm)

核心操作： 跨样本（Batch）做归一化。

具体做法： 对于同一个特征通道（Channel），它会把当前 Batch 里所有样本的特征值凑在一起，计算出均值和方差，然后进行缩放。结果是(1 * D)

Layer Normalization (LayerNorm)

核心操作： 样本内（Layer/Feature）做归一化。

具体做法： 彻底切断样本之间的联系。对于某一个特定样本，它把该样本当前层的所有特征通道（Channel）凑在一起，计算这个单一样本自身的均值和方差。结果是(N * 1)

我们减去μ 除以σ,(归一化)

Group Norm, Instance Norm

对一个样本内不同的一个或者几个样本通道做归一化

Dropout

在每次反向传播中，我们会将某些激活函数置零，让某些点对最后结果不起作用

redundent representation : 让分类器不会过度依赖某一个特征，让它泛化在更多特征上

丢弃率: 50% || 25%

会让train时间 longer , test 更准确

注意，如果dropout 50%的数值，训练时： 随机把 $50%$ 的神经元清零，然后把剩下的神经元全部除以 $0.5$（也就是乘以 2）。

反向传播： 将上游梯度0 传入，这些数据不会被更新

或者训练时间不变，predict的时候乘以dropout率 p

激活函数

sigmoid : 不再用了

sigmoid两端的gradient非常小

原因 : 多层的sigmoid会让梯度越来越小， 最后近乎没有

ReLu: Rectified Linear Unit: 计算方便， 梯度为1不会造成梯度快速下降

问题：在 x = 0接近处不平滑

GELU： 在中间更加平滑 x * N(x) -> x 乘以正态分布

同时还有SiLU / ELU /Leaky ReLu /

Activation Function的位置：线性层后面：全连接层，卷积层

CNN Archetectures

最后的FC 1000 -> ImageNet有1000个类别

why 3 * 3 Conv(stride = 1, pad = 1) : 在每一层都可以将感受野边长加长 2 ， 3个3 * 3 Conv -> 7 * 7 感受野，且 3个 3 * 3有更少的 参数，且多个3*3有更多非线性的激活函数

RestNet

更深层模型为什么表示能力反而变差了：更难优化！(局部最小化)

我们至少想让深层模型和浅层模型一样的表示能看里

所以我们让 H(x) = F(x)(deep layer的接受x后的输出) + x(shallow layer的输出) 这就是Restnet

用了RestNet后 ： 更大层数带来了更好的性能，一直到100-152层之后就没有很大变化

在极深的网络中，反向传播时梯度非常容易在倒退回前几层时“干涸”（梯度消失）。而右边的 Identity 相当于铺设了一条梯度高速公路。在反向传播计算导数时，$\frac{d(F(X)+X)}{dX} = \frac{dF(X)}{dX} + 1$。即使主干部分的梯度 $\frac{dF(X)}{dX}$ 趋近于 $0$，后面那个恒稳的 $+1$ 也能保证梯度毫无损耗地传回前一层，从而彻底解决了网络堆叠太深时无法训练的问题。
